题目1：校园课程资料智能检索与问答服务系统
项目要素	明确要求
项目目标	面向学生和教师，建立课程资料上传、检索、问答和反馈系统，解决课件、实验指导书、作业要求分散难查的问题。
支撑知识	Java Web/Python Web、数据库、文本处理、信息检索、基础LLM调用。
基础必做模块	用户登录；课程资料上传；文档元数据管理；关键词检索；问答历史；反馈记录；管理端审核。
前沿增强模块	RAG文档切分、向量检索、答案来源引用；可选本地小模型或云端LLM；增加提示注入防护和无答案拒答。
准备阶段	收集5门课程的公开资料；完成文档分类、字段设计、脱敏处理；确定LLM或向量库工具。
实现阶段	实现文档上传解析、chunk切分、向量化入库、检索排序、LLM生成、引用展示、用户反馈和日志记录。
结果展示	现场上传一份新资料，完成索引更新；输入3个问题，展示答案、引用片段、历史记录和错误反馈；演示无关问题拒答。
