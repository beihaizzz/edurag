### 1.2 建表 DDL

#### 1.2.1 用户表

```sql
CREATE TABLE users (
    id              SERIAL PRIMARY KEY,
    username        VARCHAR(64)  NOT NULL UNIQUE,           -- 学号(学生) / 工号(教师/admin)
    password_hash   VARCHAR(128) NOT NULL,              -- bcrypt 输出固定 60 字符，128 留有升级空间
    role            VARCHAR(16)  NOT NULL CHECK (role IN ('student', 'teacher', 'admin')),
    real_name       VARCHAR(64)  NOT NULL DEFAULT '',
    email           VARCHAR(128) DEFAULT NULL,
    force_password_change BOOLEAN NOT NULL DEFAULT FALSE,  -- 管理员重置后强制修改密码
    is_active       BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at      TIMESTAMP    NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP    NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_role     ON users(role);
```

#### 1.2.2 课程表

```sql
CREATE TABLE courses (
    id              SERIAL PRIMARY KEY,
    name            VARCHAR(256) NOT NULL,
    semester        VARCHAR(32)  NOT NULL
                        CHECK (semester ~ '^\d{4}-\d{4}-[12]$'),  -- e.g. '2025-2026-2'
    teacher_id      INTEGER      NOT NULL REFERENCES users(id),
    description     TEXT         DEFAULT '',
    is_deleted      BOOLEAN      NOT NULL DEFAULT FALSE,   -- 软删除标记
    created_at      TIMESTAMP    NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_courses_teacher ON courses(teacher_id);
CREATE INDEX idx_courses_semester ON courses(semester);
CREATE INDEX idx_courses_deleted ON courses(is_deleted) WHERE is_deleted = FALSE;
```

#### 1.2.3 文档表

```sql
CREATE TABLE documents (
    id              SERIAL PRIMARY KEY,
    course_id       INTEGER      REFERENCES courses(id) ON DELETE SET NULL,  -- 课程删除后保留文档
    uploader_id     INTEGER      NOT NULL REFERENCES users(id),
    filename        VARCHAR(512) NOT NULL,
    file_type       VARCHAR(32)  NOT NULL CHECK (file_type IN (
                        'courseware', 'lab_guide', 'assignment', 'reference', 'other'
                    )),
    title           VARCHAR(512) NOT NULL DEFAULT '',
    description     TEXT         DEFAULT '',
    tags            JSONB        DEFAULT '[]'::jsonb,  -- ["栈", "队列", "数据结构"]
    file_path       VARCHAR(1024) NOT NULL,
    file_size       BIGINT       NOT NULL DEFAULT 0,
    file_hash       VARCHAR(64),                        -- SHA-256 去重
    status          VARCHAR(16)  NOT NULL DEFAULT 'pending'
                        CHECK (status IN ('pending', 'approved', 'rejected')),
    processing_status VARCHAR(16)  NOT NULL DEFAULT 'pending'
                        CHECK (processing_status IN ('pending', 'processing', 'completed', 'failed')),
    audit_comment   TEXT         DEFAULT '',
    auditor_id      INTEGER      REFERENCES users(id),
    chunk_count     INTEGER      NOT NULL DEFAULT 0,
    created_at      TIMESTAMP    NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP    NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_documents_course   ON documents(course_id);
CREATE INDEX idx_documents_status   ON documents(status);
CREATE INDEX idx_documents_uploader ON documents(uploader_id);
CREATE INDEX idx_documents_type     ON documents(file_type);
-- 关键词检索采用 ILIKE 模糊匹配（中文场景下 PG simple 分词无效），
-- 数据量小时无需额外索引；增长后可用 pg_trgm GIN 索引加速
-- CREATE INDEX idx_documents_trgm ON documents USING GIN (title gin_trgm_ops);
```

#### 1.2.4 分块表（元数据，向量存 ChromaDB）

```sql
CREATE TABLE chunks (
    id              SERIAL PRIMARY KEY,
    document_id     INTEGER      NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    chunk_index     INTEGER      NOT NULL,             -- 在文档内的序号
    content         TEXT         NOT NULL,
    char_count      INTEGER      NOT NULL DEFAULT 0,   -- 字符数（精确 token 需 tiktoken）
    chroma_id       VARCHAR(128) NOT NULL,             -- 对应 ChromaDB 中的 document id
    created_at      TIMESTAMP    NOT NULL DEFAULT NOW(),

    UNIQUE (document_id, chunk_index)
);

CREATE INDEX idx_chunks_document ON chunks(document_id);
```

#### 1.2.5 对话会话索引表（替代原 qa_history）

对话内容由 LangGraph checkpointer 持久化，本表仅作为轻量索引，用于"列出用户的所有对话会话"。

```sql
CREATE TABLE user_sessions (
    thread_id       VARCHAR(64)  PRIMARY KEY,          -- LangGraph thread_id
    user_id         INTEGER      NOT NULL REFERENCES users(id),
    course_id       INTEGER      REFERENCES courses(id) ON DELETE SET NULL,
    first_question  VARCHAR(500) NOT NULL,              -- 首条问题（列表预览用）
    turn_count      INTEGER      NOT NULL DEFAULT 1,    -- 对话轮数
    created_at      TIMESTAMP    NOT NULL DEFAULT NOW(),
    last_active_at  TIMESTAMP    NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_sessions_user       ON user_sessions(user_id);
CREATE INDEX idx_sessions_course     ON user_sessions(course_id);
CREATE INDEX idx_sessions_last_active ON user_sessions(last_active_at DESC);
```

**设计说明**：
- `thread_id` 直接使用 LangGraph checkpointer 的 thread_id，作为主键，避免两套 ID 体系
- 对话内容（question, answer, sources, rejection_category 等）存储在 checkpointer 中，通过 `graph.aget_state_history(thread_id)` 查询
- 反馈表（feedback）暂不引入，后续通过 checkpointer state 扩展字段实现
- 旧 `qa_history` 表废弃，历史数据通过迁移脚本导入 checkpointer

#### 1.2.6 反馈表（暂保留，后续重构）

> 反馈功能后续通过 LangGraph checkpointer state 扩展实现。当前暂时移除此表。
```

### 1.4 refresh_tokens 表与 JWT 撤销

#### 1.4.1 建表 DDL

```sql
CREATE TABLE refresh_tokens (
    id              SERIAL PRIMARY KEY,
    user_id         INTEGER      NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    token_hash      VARCHAR(128) NOT NULL,       -- SHA-256(random 32-byte token)
    expires_at      TIMESTAMP    NOT NULL,       -- now() + 7 days
    revoked         BOOLEAN      NOT NULL DEFAULT FALSE,
    created_at      TIMESTAMP    NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_refresh_user   ON refresh_tokens(user_id);
CREATE INDEX idx_refresh_hash   ON refresh_tokens(token_hash);
CREATE INDEX idx_refresh_expiry ON refresh_tokens(expires_at);
```

#### 1.4.2 令牌生命周期

完整的 refresh_token 生命周期流转：

```
1. 登录时创建
   random_bytes(32) → raw_token ──SHA-256──→ token_hash
   INSERT refresh_tokens (user_id, token_hash, expires_at=NOW()+7d)
   返回 raw_token 给客户端

2. 刷新时旋转
   客户端发送 raw_token → SHA-256 → 查表
   ┌─ 未找到 / revoked=TRUE / expires_at < NOW() → 401 "令牌无效或已过期"
   └─ 有效 → UPDATE revoked=TRUE (旧 token 作废)
           → 生成新 raw_token → INSERT 新行 → 返回新 token pair
   ⚠ 每次 refresh 都旋转（rotation），防止 token 泄露后被长期利用

3. 登出时吊销
   客户端发送 raw_token → SHA-256 → UPDATE revoked=TRUE

4. 管理员禁用用户时批量吊销
   is_active=FALSE → UPDATE refresh_tokens SET revoked=TRUE WHERE user_id=?

5. 定时清理过期令牌
   每日凌晨: DELETE FROM refresh_tokens WHERE expires_at < NOW() - INTERVAL '30 days'
```

> **关于 access_token**：access_token 是**无状态 JWT**，不在数据库中存储，有效期仅 1 小时。管理员禁用用户后，该用户已持有的 access_token 在 1 小时内仍然有效。对于校园课程项目，1 小时的安全窗口可接受。管理员禁用用户时同时吊销所有 refresh_token，用户无法获取新的 access_token——1 小时后自然完全锁定。

### 1.5 数据库索引设计总结

| 表 | 索引名 | 列 | 类型 | 用途 |
|----|--------|----|------|------|
| `users` | `idx_users_username` | `username` | UNIQUE (DDL 隐含) | 登录查找 / 注册去重 |
| `users` | `idx_users_role` | `role` | B-tree | 按角色筛选用户 |
| `courses` | `idx_courses_teacher` | `teacher_id` | B-tree | 教师课程列表 |
| `courses` | `idx_courses_semester` | `semester` | B-tree | 按学期筛选 |
| `courses` | `idx_courses_deleted` | `is_deleted` | Partial B-tree `WHERE is_deleted = FALSE` | 活跃课程快速查询 |
| `documents` | `idx_docs_course` | `course_id` | B-tree | 按课程查文档 |
| `documents` | `idx_docs_uploader` | `uploader_id` | B-tree | 教师文档列表 |
| `documents` | `idx_docs_status` | `(status, processing_status)` | Composite B-tree | 审核列表 / 处理状态筛选 |
| `documents` | `idx_docs_file_hash` | `file_hash` | B-tree | SHA-256 去重检测 |
| `documents` | `idx_docs_created` | `created_at DESC` | B-tree | 首页最新文档 |
| `user_sessions` | `idx_sessions_user` | `user_id` | B-tree | 用户会话列表查询 |
| `user_sessions` | `idx_sessions_course` | `course_id` | B-tree | 按课程统计会话 |
| `user_sessions` | `idx_sessions_last_active` | `last_active_at DESC` | B-tree | 会话时间排序 |
| `chunks` | `idx_chunks_document` | `(document_id, chunk_index)` | Composite B-tree | 按文档 ID + 序号查 chunk |
| `feedback` | `idx_feedback_qa` | `qa_id` | B-tree | 按问答查反馈 |
| `feedback` | `idx_feedback_type` | `type` | B-tree | 按类型统计（helpful/useless/wrong） |
| `audit_logs` | `idx_audit_user` | `user_id` | B-tree | 按用户查日志 |
| `audit_logs` | `idx_audit_created` | `created_at DESC` | B-tree | 日志时间排序 |
| `audit_logs` | `idx_audit_action` | `action` | B-tree | 按操作类型筛选 |
| `refresh_tokens` | `idx_refresh_user` | `user_id` | B-tree | 按用户批量吊销 |
| `refresh_tokens` | `idx_refresh_hash` | `token_hash` | B-tree | 令牌查找（精确匹配） |
| `refresh_tokens` | `idx_refresh_expiry` | `expires_at` | B-tree | 过期清理 |

> **索引策略说明**：全部使用 B-tree（PostgreSQL 默认）。不创建全文检索索引——本项目关键词检索使用 ILIKE 模糊匹配（`pg_trgm` 扩展自动加速 `LIKE '%keyword%'`）。Composite 索引符合最左前缀优化，如 `(status, processing_status)` 既可加速 `WHERE status='approved'` 也可加速 `WHERE status='approved' AND processing_status='completed'`。

### 1.6 Redis 缓存键设计

> **定位**: Redis 为**可选组件**。最小化部署（单机）可跳过 Redis，所有缓存功能优雅降级。配置文件 `REDIS_URL` 为空时禁用 Redis。

**键命名规范**: `edurag:{namespace}:{entity}:{identifier}`

| Key Pattern | 类型 | TTL | 说明 |
|-------------|------|-----|------|
| `edurag:session:{username}:login_attempts` | String | 15 min | 登录失败计数器，≥5 次触发锁定 |
| `edurag:session:{username}:locked` | String | 15 min | 登录锁定标记，前端提示"请15分钟后再试" |
| `edurag:cache:user:{id}` | Hash | 5 min | `{id, username, role, real_name}` 缓存，减少 DB 查询 |
| `edurag:cache:course:{id}` | Hash | 10 min | 课程信息缓存 |
| `edurag:cache:doc:{id}` | Hash | 10 min | 文档元信息缓存 |
| `edurag:cache:search:{sha256(query+params)}` | List | 5 min | 相同查询参数的结果缓存（去重） |
| `edurag:rate:{username}:{endpoint}` | String | 1 min | 接口频率限制计数器（10 req / min / user） |
| `edurag:queue:doc_process` | List | — | 文档处理队列（预留，当前使用 BackgroundTasks） |

**降级策略（Redis 不可用时）**：
- 登录限流：降级为进程内存字典（`collections.defaultdict`），不保证多 Worker 一致性
- 数据缓存：全部跳过，每次直接查 PostgreSQL
- 频率限制：降级为进程内存计数器
- 消息队列：不启用（使用 BackgroundTasks）

### 1.7 软删除与数据一致性

| 实体 | 删除策略 | 具体行为 |
|------|----------|----------|
| **课程 (courses)** | 软删除 | `is_deleted = TRUE`，所有查询需添加 `WHERE is_deleted = FALSE`（通过 SQLAlchemy 全局 filter 或 Repository 层封装） |
| **课程关联文档** | 保留 | 课程删除后 `documents.course_id` 设为 `NULL`（`ON DELETE SET NULL`），文档本身不丢失 |
| **文档 (documents)** | 物理删除 | Cascade 删除所有关联 chunks（PostgreSQL `ON DELETE CASCADE`）+ ChromaDB.delete(where={document_id}) + 删除本地文件 |
| **文档关联 chunks** | 级联删除 | PostgreSQL 外键 `ON DELETE CASCADE`，同时清理 ChromaDB 向量数据 |
| **问答记录 (qa_history)** | 软删除 | 设置 `deleted_at TIMESTAMP` 标记，物理保留用于统计和分析 |
| **反馈 (feedback)** | 物理删除 | 直接 DELETE（同一用户可重新提交反馈；`uq_feedback_user_qa` 约束物理删除后消除） |
| **审计日志 (audit_logs)** | 永久保留 | 不提供删除接口，仅通过 `created_at` 索引加速时间范围查询 |
| **refresh_tokens** | 过期自动清理 | 每日定时任务清理 `expires_at < NOW() - 30 days` 的已过期行 |

> **查询约定**: 所有 Repository 层查询方法默认包含软删除过滤。示例：
> ```python
> # backend/app/repositories/course_repo.py
> async def find_all(self, **filters) -> list[Course]:
>     stmt = select(Course).where(Course.is_deleted == False)
>     ...
> ```
