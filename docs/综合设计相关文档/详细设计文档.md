## 1. 核心模块详细设计

### 1.1 文档解析与 Chunk 切分模块

#### 1.1.1 文档解析器设计

使用 **Docling** 作为主解析引擎，统一处理 PDF/DOCX/PPTX/TXT 等多格式文档，输出结构化 Markdown。PyMuPDF 作为纯文本 PDF 的快速兜底。

```python
# backend/app/utils/parser.py

from docling.document_converter import DocumentConverter

class DocumentParser:
    """文档解析器 — 基于 Docling 统一入口"""
    
    def __init__(self):
        self.converter = DocumentConverter()
    
    def parse(self, file_path: str) -> str:
        """解析任意支持格式的文档，返回 Markdown"""
        result = self.converter.convert(file_path)
        return result.document.export_to_markdown()
    
    def parse_with_structure(self, file_path: str) -> dict:
        """解析并返回结构化信息（含表格、阅读顺序等）"""
        result = self.converter.convert(file_path)
        doc = result.document
        return {
            "markdown": doc.export_to_markdown(),
            "tables": [t.export_to_dataframe() for t in doc.tables],
            "page_count": len(doc.pages) if doc.pages else 0,
        }


# 支持的格式（Docling 原生）:
# .pdf, .docx, .pptx, .xlsx, .html, .md, .txt, 图片 (PNG/JPEG)
# 无需手动维护解析器注册表，Docling 自动识别文件类型
```

**解析策略**：
- **Docling** 作为主解析：MIT 许可证，IBM 维护，表格精度 97.9%，内置 LangChain 集成
- **PyMuPDF** 作为纯文本兜底：速度 0.01s/页，当 Docling 首次加载模型时可用作热备

#### 1.1.2 Chunk 切分策略

```python
# backend/app/utils/chunker.py

from langchain_text_splitters import RecursiveCharacterTextSplitter

class DocumentChunker:
    """文档语义切分器"""
    
    def __init__(
        self,
        chunk_size: int = 800,      # 每块最大字符数（中文约 400-500 tokens）
        chunk_overlap: int = 150,   # 块之间重叠字符数
        separators: list[str] = None
    ):
        self.chunk_size = chunk_size
        self.chunk_overlap = chunk_overlap
        # 分隔符优先级: 段落 → 句子 → 短语
        self.separators = separators or [
            "\n\n",    # 空行（段落分隔）
            "\n",      # 换行
            "。",      # 中文句号
            "；",      # 中文分号
            "，",      # 中文逗号
            ".",       # 英文句号
            " ",       # 空格
            "",        # 逐字切分（兜底）
        ]
        self._splitter = RecursiveCharacterTextSplitter(
            chunk_size=chunk_size,
            chunk_overlap=chunk_overlap,
            separators=self.separators,
            keep_separator=True,     # 保留分隔符在 chunk 末尾
        )
    
    def split(self, text: str) -> list[str]:
        """将文本切分为语义块"""
        chunks = self._splitter.split_text(text)
        # 过滤空块和过短块
        return [c.strip() for c in chunks if len(c.strip()) > 20]
    
    def split_with_metadata(
        self, text: str, document_id: int
    ) -> list[dict]:
        """切分并附加元数据"""
        chunks = self.split(text)
        result = []
        for i, chunk in enumerate(chunks):
            result.append({
                "document_id": document_id,
                "chunk_index": i,
                "content": chunk,
                "char_count": len(chunk),  # 字符数（精确 token 数需 tiktoken 计算）
            })
        return result
```

**切分参数调优建议**：
- `chunk_size=800`：中文约 400-500 tokens，适合 bge-m3 的 8192 token 上下文窗口；本地 bge-large-zh-v1.5 兜底时仍适配其 512 token 最佳窗口
- `chunk_overlap=150`：确保上下文连贯，避免关键信息被截断
- 对于 PPT 转文本（段落短、标题多），可降低 `chunk_size` 至 500

### 1.2 向量化与检索引擎模块

#### 1.2.1 Embedding 服务

采用 **双 Provider 抽象** 设计，统一 `BaseEmbedding` 接口，支持运行时切换：

- **SiliconFlow API**（推荐）：调用 `BAAI/bge-m3` (1024d)，OpenAI 兼容协议，无需本地 GPU
- **本地 sentence-transformers**（兜底）：`bge-large-zh-v1.5` (1024d)，离线/内网环境使用

```python

_embedding_instance: BaseEmbedding | None = None


def get_embedding() -> BaseEmbedding:

    if _embedding_instance is None:
```

**Provider 选择逻辑**：

| 场景 | 推荐 Provider | 理由 |
|------|:---:|------|
| 生产环境（有网络） | `siliconflow` | 无需 GPU，零部署，bge-m3 多语言效果优于 bge-large-zh |
| 离线/内网环境 | `local` | 本地推理，无外部依赖 |
| 开发调试 | `siliconflow` | 快速启动，无需下载 GB 级模型 |

#### 1.2.2 关键词检索服务

```python
# backend/app/services/search_service.py

from sqlalchemy import select, func, and_, or_, text
from sqlalchemy.ext.asyncio import AsyncSession

class SearchService:
    """检索服务"""
    
    def __init__(self, db: AsyncSession, vector_store):   # vector_store: VectorStoreService
        self.db = db
        self.vector_store = vector_store
    
    async def keyword_search(
        self, query: str, course_id: int = None,
        doc_type: str = None, page: int = 1, page_size: int = 10
    ) -> dict:
        """基于 ILIKE 模糊匹配的关键词搜索

        NOTE: 中文场景下 PostgreSQL 默认 'simple' 分词（按空格切分）对中文无效。
        此处选用 ILIKE 做模糊匹配。生产环境如流量增长可升级为：
        - 安装 pg_jieba / zhparser 中文分词扩展，启用全文检索
        - 或使用 Elasticsearch + ik 分词器
        """
        like_pattern = f"%{query}%"

        # 构建 WHERE 条件（全部使用参数化，无 f-string 拼接）
        conditions = [Document.status == 'approved']
        if course_id is not None:
            conditions.append(Document.course_id == course_id)
        if doc_type is not None:
            conditions.append(Document.file_type == doc_type)

        # ILIKE 匹配：标题 OR 描述 OR chunk 内容
        match_cond = or_(
            Document.title.ilike(like_pattern),
            Document.description.ilike(like_pattern),
            Chunk.content.ilike(like_pattern),
        )
        conditions.append(match_cond)

        where_clause = and_(*conditions)

        # ── COUNT 查询（单独执行，获取总记录数）──
        count_query = (
            select(func.count(func.distinct(Document.id)))
            .select_from(Document)
            .outerjoin(Chunk, Chunk.document_id == Document.id)
            .join(Course, Document.course_id == Course.id)
            .where(where_clause)
        )
        total_result = await self.db.execute(count_query)
        total = total_result.scalar_one()

        # ── 主查询：文档 + 匹配的 chunk 片段 ──
        # 用 DISTINCT 去重（一个文档可能多个 chunk 匹配）
        main_query = (
            select(
                Document.id,
                Document.title,
                Document.file_type,
                Document.file_path,
                Course.name.label("course_name"),
                Chunk.content.label("snippet"),
            )
            .select_from(Document)
            .join(Course, Document.course_id == Course.id)
            .outerjoin(Chunk, Chunk.document_id == Document.id)
            .where(where_clause)
            .order_by(Document.id.desc())
            .offset((page - 1) * page_size)
            .limit(page_size)
        )
        result = await self.db.execute(main_query)
        rows = result.fetchall()

        return {
            "results": [dict(r._mapping) for r in rows],
            "total": total,
            "page": page,
            "page_size": page_size,
        }
```

#### 1.3.2 Nodes 详细设计

> 以下列出的为各 Node 的核心逻辑伪代码，完整实现见 `app/graph/nodes/`。

| Node | 类型 | 输入 → 输出 | 说明 |
|------|------|------------|------|
| `classify_intent` | LLM | `question` → `intent` | DeepSeek, temperature=0, max_tokens=10，跨语言分类 |
| `rag_search` | 函数 | `question`, `course_id` → `internal_results`, `has_internal_results` | 调用 vector_store.search()，阈值 0.45 |
| `build_context` | 函数 | `internal_results` → `context`, `sources`, `search_mode="internal"` | 拼接 `[来源N: 标题]\n内容` |
| `web_search` | API+函数 | `question` → `context`, `sources`, `has_web_results`, `search_mode="web"` | Tavily API, 10s 超时, 异常返回 has_web_results=False |
| `generate_answer` | LLM | `context`, `question`, `chat_history` → `answer` | DeepSeek, temperature=0.3, 感知对话历史 |
| `review_output` | 函数+LLM | `answer`, `sources` → `review_result`, `matched_sources` | 机械引用校验 + LLM 合规审查 |
| `reject` | 函数 | — → `is_rejected=True`, `rejection_reason`, `rejection_category` | 4 种分类话术 |
| `return_answer` | 函数 | — → `is_rejected=False`, `chat_history` 追加 | 追加本轮对话 |

```python
# app/graph/edges/routing.py

def route_after_intent(state: RAGState) -> str:
    return "reject" if state["intent"] != "NORMAL" else "rag_search"

def route_after_rag(state: RAGState) -> str:
    if state["has_internal_results"]:
        return "build_context"
    if state["use_web_search"]:
        return "web_search"
    return "reject"

def route_after_web(state: RAGState) -> str:
    return "generate_answer" if state["has_web_results"] else "reject"
```

#### 1.3.8 Graph 组装

```python
# app/graph/builder.py
from langgraph.graph import StateGraph, END, START
from langgraph.checkpoint.postgres.aio import AsyncPostgresSaver

async def build_rag_graph():
    checkpointer = AsyncPostgresSaver.from_conn_string(settings.DATABASE_URL)
    await checkpointer.setup()
    builder = StateGraph(RAGState)
    # 注册 8 个节点
    builder.add_node("classify_intent", classify_intent)
    builder.add_node("rag_search", rag_search)
    builder.add_node("build_context", build_context)
    builder.add_node("web_search", web_search)
    builder.add_node("generate_answer", generate_answer)
    builder.add_node("review_output", review_output)
    builder.add_node("reject", reject)
    builder.add_node("return_answer", return_answer)
    # 固定边
    builder.add_edge(START, "classify_intent")
    builder.add_edge("build_context", "generate_answer")
    builder.add_edge("generate_answer", "review_output")
    # 条件边
    builder.add_conditional_edges("classify_intent", route_after_intent)
    builder.add_conditional_edges("rag_search", route_after_rag)
    builder.add_conditional_edges("web_search", route_after_web)
    builder.add_conditional_edges("review_output",
        lambda s: "return_answer" if s["review_result"] == "PASS" else "reject")
    builder.add_edge("return_answer", END)
    builder.add_edge("reject", END)
    return builder.compile(checkpointer=checkpointer)
```

#### 1.3.9 持久化方案

- **Graph 状态**：`AsyncPostgresSaver` checkpointer，按 `thread_id` 组织，支持多轮对话记忆
- **会话索引**：`user_sessions` 表（`thread_id PK`, `user_id`, `first_question`, `turn_count`）
- **历史查询**：`graph.aget_state_history(thread_id)`，过滤 `snapshot.next == ()` 取每轮终端状态
- **API 层**：`POST /qa` → `graph.ainvoke()` → 更新/创建 `user_sessions` → 返回 `final_state` 用户可见字段

### 1.4 安全防护模块（三层防护）

#### 1.4.1 L0 层：快速正则过滤

保留提示注入正则匹配（`_INJECTION_RE`），扩展作弊关键词模式（"满分答案""题库""直接告诉我"等）。命中 → 直接拒答，0 LLM 成本。

#### 1.4.2 L1 层：LLM 意图分类

即 §1.3.3 的 `classify_intent` 节点。每个学生问题分类为 NORMAL/CHEATING/SENSITIVE/ATTACK，非 NORMAL 直接路由到 REJECT。temperature=0, max_tokens=10，成本 ~¥0.00003/次。

#### 1.4.3 L3 层：输出审查

即 §1.3.5 的 `review_output` 节点。两步校验：① 机械引用校验（正则提取 [来源N]，筛出幻觉引用）② LLM 语义审查（检查是否直接给出完整作业/考试答案）。

#### 1.4.4 文件上传安全

保留文件类型白名单（`.pdf`, `.docx`, `.txt`, `.md`, `.pptx`）、MIME 校验、SHA-256 去重、文件大小限制（50MB）、数据脱敏（手机号/身份证/邮箱正则）。
