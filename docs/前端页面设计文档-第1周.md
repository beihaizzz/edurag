## 3. 页面详细设计

### 3.1 LoginPage（✅ 无需改动）

**布局**: 居中卡片（400px 宽），浅灰背景 `#f0f2f5`

```
 ┌──────────────────────────────┐
 │         EduRAG               │
 │  校园课程资料智能检索与问答    │
 │                              │
 │  ┌──────────────────────────┐│
 │  │  👤 学号/工号            ││
 │  │  🔒 密码                 ││
 │  │  [      登录      ]      ││
 │  └──────────────────────────┘│
 │     还没有账号？ 立即注册     │
 └──────────────────────────────┘
```

**交互逻辑**:
- 登录成功 → 如 `force_password_change=true` → 跳转 `/change-password`
- 登录成功 → 否则 → 跳转 `/{role}`（`/student` / `/teacher` / `/admin`）
- 登录失败 → 红色提示错误信息

---

### 3.2 RegisterPage（✅ 无需改动）

**布局**: 同 LoginPage，标题改为"注册"，副标题"使用学号/工号注册"

**交互逻辑**:
- 注册成功 → 提示"注册成功，请登录" → 跳转 `/login`
- 密码至少 6 位

---

### 3.3 ChangePasswordPage（✅ 无需改动）

**布局**: 同 LoginPage 居中卡片

**交互逻辑**:
- `force_password_change=true` → 仅显示新密码框 → 调用 `resetPassword` → 跳转 `/login`
- `force_password_change=false` → 显示旧密码 + 新密码框 → 调用 `changePassword` → 跳转 `/{role}`

## 4. 交互流程

### 4.1 登录后角色分流

```
用户登录 → force_password_change?
  ├── Yes → /change-password → 修改密码 → /login（重新登录）
  └── No  → role?
            ├── student → /student
            ├── teacher → /teacher
            └── admin   → /admin
```

### 4.2 Layout 导航

```
Layout
├── Sider (左侧导航)
│   ├── Logo (EduRAG, 点击回首页)
│   └── Menu (根据 role 动态渲染)
│       ├── student: [首页检索, 搜索结果, 问答历史]
│       ├── teacher: [首页, 上传文档, 文档管理]
│       └── admin:   [仪表盘, 数据统计, 文档审核, 用户管理]
│
├── Header (顶部栏)
│   ├── 用户名 + 角色标签
│   └── 用户头像 Dropdown → [修改密码, 退出登录]
│
└── Content (主内容区, margin: 24px)
    └── <Outlet />
```

### 4.3 路由守卫

- 未登录访问任何需登录路由 → 重定向 `/login`
- student 访问 `/teacher/*` → 重定向 `/student`
- teacher 访问 `/admin/*` → 重定向 `/teacher`

## 5. 技术规范

### 5.1 组件组织

```
src/
├── components/          # 通用组件（跨角色复用）
│   ├── Layout.tsx       # 主布局 (Sider + Header + Content)
│   ├── AuthGuard.tsx    # 路由守卫
│   └── SearchBar.tsx    # 搜索框
├── pages/
│   ├── auth/            # 认证相关页面
│   │   ├── LoginPage.tsx
│   │   ├── RegisterPage.tsx
│   │   └── ChangePasswordPage.tsx
│   ├── student/         # 学生端页面
│   │   ├── StudentHomePage.tsx
│   │   ├── SearchResults.tsx
│   │   ├── QAHistory.tsx
│   │   └── QADetail.tsx
│   ├── teacher/         # 教师端页面
│   │   ├── TeacherHomePage.tsx
│   │   ├── UploadDocument.tsx
│   │   └── DocumentManage.tsx
│   └── admin/           # 管理端页面
│       ├── AdminHomePage.tsx
│       ├── Dashboard.tsx
│       ├── ReviewList.tsx
│       └── UserManage.tsx
├── stores/              # Zustand 状态管理
│   └── authStore.ts     # 认证状态（已实现）
├── services/            # API 请求封装
│   ├── api.ts           # Axios 实例 + 拦截器（已实现）
│   └── authApi.ts       # 认证 API（已实现）
├── hooks/               # 自定义 Hook
│   └── useAuth.ts       # 认证 Hook
├── types/               # TypeScript 类型
│   └── api.ts           # API 响应类型（已实现）
└── router/
    └── index.tsx        # 路由配置（已实现）
```

### 5.2 样式约定

- **UI 框架**: Ant Design 5，中文 locale
- **布局**: Flex + Grid，无额外 CSS 框架
- **色彩**: Ant Design 默认主题色 `#1677ff`
- **响应式**: Sider `breakpoint="lg"` → 小屏收起

### 5.3 状态管理

- **认证状态**: Zustand + persist (localStorage, key: `auth-storage`)
- **其他状态**: 页面内 useState，不做全局缓存（第 2 周按需扩展）
